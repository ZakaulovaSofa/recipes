# Архитектура проекта

Проект использует комбинированную архитектуру:

- серверный HTML-интерфейс на Flask + Jinja2;
- отдельный REST API слой с префиксом `/api`;
- единая база данных и ORM-модели.

HTML routes используются пользователями сайта.
API routes предназначены для программного доступа к данным и демонстрации REST-подхода.
