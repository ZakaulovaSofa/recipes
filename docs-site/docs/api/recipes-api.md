# REST API для сущности Recipe

## Получение списка рецептов

**Endpoint:** `GET /api/recipes`

По умолчанию возвращаются опубликованные рецепты со статусом published.

Поддерживаются query-параметры:

- limit – Количество рецептов
- offset – Смещение для пагинации
- status – Фильтр по статусу рецепта

**Пример:**

`GET /api/recipes?limit=10&offset=0&status=published` – Получение одного рецепта

**Endpoint:** `GET /api/recipes/{recipe_id}`

Возвращает детальную информацию о рецепте, включая:

- id;
- название;
- описание;
- статус;
- изображение; 
- ингредиенты; 
- автора; 
- даты создания и обновления; 
- шаги приготовления.

## Создание рецепта

**Endpoint:** `POST /api/recipes`

Требует авторизации.

Данные передаются в теле запроса в формате JSON.

**Пример:**

```json
{
  "title": "Омлет с сыром",
  "description": "Простой завтрак из яиц и сыра.",
  "ingredients": [
    {
      "name": "Яйцо",
      "amount": "2",
      "unit": "шт"
    }
  ],
  "steps": [
    {
      "text": "Взбить яйца."
    }
  ]
}
```

После создания рецепт получает статус pending, потому что он должен пройти модерацию.

## Обновление рецепта

**Endpoint:** `PATCH /api/recipes/{recipe_id}`

Требует авторизации.

Пользователь может обновить только собственный рецепт.

**Пример:**
```json
{
  "title": "Омлет с сыром и зеленью"
}
```

После обновления статус рецепта снова становится pending.

## Удаление рецепта

**Endpoint:** `DELETE /api/recipes/{recipe_id}`

Требует авторизации.

Пользователь может удалить только собственный рецепт.

## Статусы рецептов

Рецепт может иметь один из трёх статусов:

- published – рецепт опубликован
- pending – рецепт находится на модерации
- rejected – рецепт отклонён

## Пагинация

Список рецептов возвращается с объектом `pagination`.

Пример:
```json
{
  "pagination": {
    "limit": 20,
    "offset": 0,
    "total": 35,
    "has_next": true,
    "has_prev": false
  }
}
```

Это позволяет клиенту понимать, есть ли следующая или предыдущая страница.

## OpenAPI-спецификация

Для API рецептов подготовлена OpenAPI-спецификация. Она описывает:

- доступные endpoint-ы;
- HTTP-методы;
- параметры;
- JSON-схемы запросов;
- JSON-схемы ответов;
- возможные HTTP-статусы;
- примеры успешных и ошибочных ответов.

OpenAPI используется для генерации красивой документации API в Docusaurus.
