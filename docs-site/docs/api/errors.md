# Ошибки API

REST API проекта Be cooking возвращает ошибки в формате JSON. Это позволяет API-клиенту программно обрабатывать
неуспешные ответы и показывать пользователю понятные сообщения.

## Общий формат ошибки

Все ошибки API имеют единую структуру:

```json
{
    "success": false,
    "error": "Описание ошибки"
}
```

Поле success показывает, что операция завершилась неуспешно.
Поле error содержит текстовое описание проблемы.

## Вспомогательная функция ошибок

В коде используется единая функция для формирования ошибочных ответов:

```python
def api_error(message, status_code):
    return jsonify({
        "success": False,
        "error": message
    }), status_code
```

Это помогает поддерживать одинаковый формат ошибок во всех API endpoint-ах.

## Используемые HTTP-статусы

- `200 OK` | Успешный запрос
- `201 Created` | Объект создан
- `400 Bad Request` | Ошибка запроса
- `401 Unauthorized` | Требуется авторизация
- `403 Forbidden` | Нет прав
- `404 Not Found` | Объект не найден

## 400 Bad Request

Статус 400 возвращается, если клиент отправил некорректные данные.

Примеры ситуаций:

- тело запроса не является JSON-объектом;
- отсутствует обязательное поле;
- поле имеет неверный тип;
- массив ингредиентов пустой;
- массив шагов приготовления пустой;
- передан некорректный статус рецепта;
- параметр limit выходит за допустимые границы;
- параметр offset отрицательный.

**Пример ответа:**

```json
{
    "success": false,
    "error": "Поле title должно быть непустой строкой."
}
```

## 401 Unauthorized

Статус 401 возвращается, когда пользователь пытается выполнить действие, требующее авторизации, но не вошёл в систему.

Например: `POST /api/recipes` без активной сессии вернёт:
```json
{
"success": false,
"error": "Необходима авторизация."
}
```

Этот статус используется для операций:

- создание рецепта;
- редактирование рецепта;
- удаление рецепта.

## 403 Forbidden

Статус 403 возвращается, когда пользователь авторизован, но не имеет прав на действие.

Например, пользователь пытается изменить рецепт другого пользователя: `PATCH /api/recipes/15`

**Ответ:**

```json
{
"success": false,
"error": "Нет прав на изменение этого рецепта."
}
```

## 404 Not Found

Статус 404 возвращается, если объект не найден.

Например: `GET /api/recipes/9999`

**Ответ:**

```json
{
"success": false,
"error": "Рецепт не найден."
}
```

Этот статус используется, если рецепт с указанным recipe_id отсутствует в базе данных.

## Ошибки валидации рецепта

Для создания и обновления рецептов используется валидация входного JSON.

### Обязательные поля при создании

Для `POST /api/recipes` обязательны поля:

- title;
- description;
- ingredients;
- steps.

Если одно из этих полей отсутствует, API возвращает 400.

**Пример:**
```json
{
"success": false,
"error": "Обязательное поле отсутствует: description."
}
```

### Поле title

Поле title должно быть непустой строкой и не должно быть длиннее 200 символов.

**Ошибка:**

```json
{
"success": false,
"error": "Поле title должно быть непустой строкой."
}
```

### Ошибка при слишком длинном названии:

```json
{
"success": false,
"error": "Поле title не должно быть длиннее 200 символов."
}
```

### Поле description

Поле description должно быть непустой строкой.

**Ошибка:**

```json
{
"success": false,
"error": "Поле description должно быть непустой строкой."
}
```

### Поле ingredients

Поле ingredients должно быть массивом.

Каждый ингредиент должен быть JSON-объектом с полями:

- name;
- amount;
- unit.

**Пример ошибки:**

```json
{
"success": false,
"error": "Поле ingredients должно быть массивом."
}
```

**Если ингредиенты не указаны:**

```json
{
"success": false,
"error": "Нужно указать хотя бы один ингредиент."
}
```

**Если у ингредиента нет названия:**

```json
{
"success": false,
"error": "У каждого ингредиента должно быть непустое поле name."
}
```

### Поле steps

Поле steps должно быть массивом.

Каждый шаг должен быть JSON-объектом с обязательным полем text.

**Если шаги не указаны:**

```json
{
"success": false,
"error": "Нужно указать хотя бы один шаг приготовления."
}
```

**Если текст шага пустой:**

```json
{
"success": false,
"error": "У каждого шага должно быть непустое поле text."
}
```

## Итог

API Be cooking использует единый формат ошибок и понятные HTTP-статусы. Это делает поведение API предсказуемым, упрощает
отладку и позволяет удобно демонстрировать обработку ошибок при проверке проекта.
