# REST API

В проекте Be cooking реализован отдельный REST API слой для работы с данными в формате JSON.

Основной пользовательский интерфейс приложения построен на серверном рендеринге Flask и Jinja2-шаблонов. Пользователь работает со страницами сайта: рецептами, профилем, корзиной, избранным, комментариями и административными разделами.

REST API существует как отдельный программный интерфейс с префиксом `/api`. Он предназначен для получения и изменения данных в формате JSON.

## Зачем в проекте нужен REST API

REST API нужен для:

- выполнения требований проекта по работе с JSON;
- демонстрации ресурсно-ориентированной архитектуры;
- возможности программного доступа к данным;
- потенциальной интеграции с внешним frontend-приложением;
- будущего расширения проекта.

## Гибридная архитектура

В проекте используется комбинированный подход:

```text
HTML routes → страницы для пользователей
API routes  → JSON-интерфейс для программного доступа
```

Пример:

- `GET /recipes/1` – возвращает HTML-страницу рецепта.
- `GET /api/recipes/1` – возвращает JSON-объект рецепта.

Оба маршрута используют одну и ту же базу данных и ORM-модели, но возвращают данные в разном формате.


## Основной ресурс API

На текущем этапе API реализован для сущности рецепта.

Рецепт является центральной сущностью проекта, поэтому для него реализованы основные CRUD-операции:

- `GET /api/recipes` |	Получить список рецептов
- `GET /api/recipes/{recipe_id}` |	Получить один рецепт
- `POST	/api/recipes` | Создать рецепт
- `PATCH /api/recipes/{recipe_id}` | Обновить рецепт
- `DELETE /api/recipes/{recipe_id}` | Удалить рецепт

## Использование HTTP-методов

API использует HTTP-методы по смыслу:

- GET – Получение данных
- POST – Создание новой записи
- PATCH – Частичное обновление существующей записи
- DELETE – Удаление записи

Такой подход соответствует REST-стилю проектирования API.

## Формат данных

API использует JSON как основной формат обмена данными.

Ответы формируются через jsonify.

Пример успешного ответа:

```json
{
  "success": true,
  "item": {
    "id": 1,
    "title": "Омлет с сыром",
    "description": "Простой завтрак из яиц и сыра.",
    "status": "published"
  }
}
```


Ошибки также возвращаются в JSON-формате:

```json
{
  "success": false,
  "error": "Рецепт не найден."
}
```


## Итог

REST API в Be cooking реализует ресурсно-ориентированный JSON-интерфейс для работы с рецептами. Он дополняет основной HTML-интерфейс и показывает, как приложение может предоставлять данные не только пользователю через страницы, но и программным клиентам через JSON.
