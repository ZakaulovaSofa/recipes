# E2E-тесты системы

В рамках практики для тестирования системы был выбран формат горизонтальных E2E-тестов с описанием тест-кейсов и результатов внутри Docusaurus-документации.

E2E-тестирование проверяет работу системы с точки зрения пользователя: от начала действия до получения результата.  

В проекте Be cooking E2E-сценарии проверялись вручную через пользовательский интерфейс сайта.

## Формат описания тест-кейса

- **Сценарий**: название сценария тест-кейса
- **Предусловие**: что должно быть выполнено перед началом сценария
- **Шаги**: описание действий пользователя в рамках сценария
- **Ожидаемый результат**: нормальное поведение системы, которого мы хотим достичь
- **Статус**: пройден тест или нет

## Пример тест-кейса

### Сценарий

Добавление рецепта в избранное

### Предусловие

- Пользователь авторизован
- Рецепт опубликован
- Рецепта ещё нет в избранном пользователя

### Шаги

1. Открыть рецепт
2. Нажать на иконку закладки

### Ожидаемый результат 

Иконка становится активной, рецепт появляется в разделе “Избранные рецепты”

### Статус

Пройден
